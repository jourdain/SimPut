Hypre/BoomerAMG

hypre_coarsen - Specifies the parallel coarsening algorithm for Hypre/BoomerAMG, hypre_coarsen (string, default=FALGOUT). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values can be one of the following:

  • CLJP - Cleary-Luby-Jones-Plassman (a parallel coarsening algorithm using independent sets),
  • RUGE_STEUBEN - Classical Ruge-Steuben on each processor, no boundary treatment (not recommended!),
  • MODIFIED_RUGE_STEUBEN - Classical Ruge-Steuben on each processor, followed by a third pass, which add coarse points on the boundaries,
  • FALGOUT - Falgout coarsening (uses RUGE_STEUBEN first, followed by CLJP using the interior coarse points generated by RUGE_STEUBEN as its first independent set),
  • PMIS - PMIS-coarsening (a parallel coarsening algorithm using independent sets, generating lower complexities than CLJP, might also lead to slower convergence), or
  • HMIS - HMIS-coarsening (uses one pass RUGE_STEUBEN on each processor independently, followed by PMIS using the interior C-points generated as its first independent set).

hypre_smoother - Specifies the smoother for both the up and down cycles for Hypre/BoomerAMG, hypre_smoother (string, default=HYB_SGS). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values can be one of the following:

  • JACOBI - Jacobi,
  • SEQ_SGS - Gauss-Seidel, sequential (very slow!),
  • HYB_GS - Hybrid Gauss-Seidel or SOR, forward solve,
  • BACK_HYB_GS - Hybrid Gauss-Seidel or SOR, backward solve,
  • HYB_SGS - Hybrid symmetric Gauss-Seidel or SSOR, or
  • GE - Gaussian elimination.

hypre_smoother_dn - Specifies the smoother for the down cycles for Hypre/BoomerAMG, hypre_smoother_dn (string, default=HYB_SGS). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values can be one of the following:

  • JACOBI - Jacobi,
  • SEQ_SGS - Gauss-Seidel, sequential (very slow!),
  • HYB_GS - Hybrid Gauss-Seidel or SOR, forward solve,
  • BACK_HYB_GS - Hybrid Gauss-Seidel or SOR, backward solve,
  • HYB_SGS - Hybrid symmetric Gauss-Seidel or SSOR, or
  • GE - Gaussian elimination.

hypre_smoother_up - Specifies the smoother for the up cycles for Hypre/BoomerAMG, hypre_smoother_up (string, default=HYB_SGS). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values can be one of the following:

  • JACOBI - Jacobi,
  • SEQ_SGS - Gauss-Seidel, sequential (very slow!),
  • HYB_GS - Hybrid Gauss-Seidel or SOR, forward solve,
  • BACK_HYB_GS - Hybrid Gauss-Seidel or SOR, backward solve,
  • HYB_SGS - Hybrid symmetric Gauss-Seidel or SSOR, or
  • GE - Gaussian elimination.

hypre_smoother_co - Specifies the smoother for the coarsest level for Hypre/BoomerAMG, hypre_smoother_co (string, default=GE). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values can be one of the following:

  • JACOBI - Jacobi,
  • SEQ_SGS - Gauss-Seidel, sequential (very slow!),
  • HYB_GS - Hybrid Gauss-Seidel or SOR, forward solve,
  • BACK_HYB_GS - Hybrid Gauss-Seidel or SOR, backward solve,
  • HYB_SGS - Hybrid symmetric Gauss-Seidel or SSOR, or
  • GE - Gaussian elimination.

interp - Specifies the parallel interpolation operator for Hypre/BoomerAMG, interp (string, default=CLASSICAL). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values can be one of the following:

  • CLASSICAL - Classical modified interpolation,
  • DIRECT - Direct interpolation (with separation of weights),
  • MULTIPASS - Multipass interpolation,
  • MULTIPASS_WTS - Multipass interpolation (with separation of weights),
  • EXT+I - Extended+i interpolation,
  • EXT+I-CC - Extended+i (if no common C neighbor) interpolation,
  • STANDARD - Standard interpolation,
  • STANDARD_WTS - Standard interpolation (with separation of weights),
  • FF - FF interpolation, or
  • FF1 - FF1 interpolation.

hypre_nodal - Enable/disable nodal systems coarsening for Hypre/BoomerAMG, hypre_nodal (string, default=Off). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. Values may be one of the following:

  • On - Nodal systems coarsening, or
  • Off - Unknown-based coarsening.

trunc_factor - Specify the truncation factor for the interpolation in Hypre/BoomerAMG, trunc_factor (float, default=0.0). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG.

pmax_elements - Set the maximum elements/row for the interpolation for Hypre/BoomerAMG, pmax_elements (integer, default=0). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG.

agg_num_levels - Set number of levels of aggressive coarsening for Hypre/BoomerAMG, agg_num_levels (integer, default=0). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG.

agg_num_paths - Set number of paths for aggressive coarsening for Hypre/BoomerAMG, agg_num_paths (integer, default=1). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG.

strong_threshold - Specify the strength threshold for Hypre/BoomerAMG, strong_threshold (float, default=0.85). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG. For 2D Laplace operators 0.25 is a good value, for 3D Laplace operators 0.5 or 0.6 is a better value. For elasticity problems, a large strength threshold, such as 0.9, is often better.

max_rowsum - Set a parameter to modify the definition of strength for diagonally dominant portions of the matrix for Hypre/BoomerAMG, max_rowsum (float, default=0.9). Only effective if type=AMG and amgpc=HYPRE and hypre_type=BOOMERAMG.
