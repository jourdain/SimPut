Boundary Conditions

The specification of boundary conditions for the incompressible Navier-Stokes.

name - A descriptive boundary condition name (string, default=null). name is not used in the derived control file. A unique id for each specific boundary condition is generated underneath the cover of the web interface. While providing a descriptive name is not required, these descriptive names clearly increase usability.

type - The type of boundary condition (string, default=Scalar Dirichlet). The type must be one of the following:

  • Scalar Dirichlet - Use a scalar Dirichlet boundary conditions,
  • Velocity Dirichlet - Use a Dirichlet velocity boundary conditions,
  • Symmetry Velocity - Use a symmetry velocity boundary conditions,
  • Heat Flux - Use a heat flux boundary condition,
  • Passive Outflow - Use a passive advective condition for use at outflow boundaries,
  • Pressure Outflow - Use a pressure outflow boundary condition, or
  • User-Defined Velocity - Use a user-defined velocity boundary condition.

Note: A symmetry velocity boundary condition specifies a velocity symmetry condition in a coordinate direction on a side-set. The surface normal on the specified side-set is aligned with the coordinate direction.

Note: A passive advective condition for use at outflow boundaries provides a mechanism to suppress artificial re-entrant flow conditions when the outflow boundary is not normal to the primary flow direction.

Note: A pressure outflow boundary condition is typically applied at outflow boundaries where the variation in pressure due to vortical flow structures is large. A pressure outflow boundary condition applies an extrapolated pressure as a traction force on the momentum equations to avoid large pressure jumps at an outflow boundary.

Note: User-defined velocity boundary conditions, within Hydra-TH, use two user-to-edit functions: setUserVel and setUserAcc in the source code file /code/src/FVM/CCIncNavierStokes/UserFunctions/UserVelBC.C, which define a formula for the velocity vector and a formula for the acceleration vector respectively. The functions setUserVel and setUserAcc have access to the mesh class, the material properties, the edge ID, the edge center coordinates, and the time.

Scalar Dirichlet Boundary Conditions

scalar - The scalar (string, default=pressure). The scalar must be one of the following:

  • Enthalpy - a scalar Dirichlet boundary condition for enthalpy, h (aka enthalpybc, ebc, enthalpy).
  • Turbulent Kinetic Energy - a scalar Dirichlet boundary condition for turbulent kinetic energy, k (aka tkebc, tke).
  • Turbulent Dissipation - a scalar Dirichlet boundary condition for turbulent dissipation, ε (aka epsbc, eps).
  • Distance Function - a scalar Dirichlet boundary condition for distance function (aka distancebc, dist, distance).
  • Hydrodynamic Pressure - a scalar Dirichlet boundary condition for hydrodynamic pressure, p (aka pressurebc, pbc, pressure).
  • Temperature - a scalar Dirichlet boundary condition for temperature, T (aka temperaturebc, tbc, temperature).
  • Turbulent Viscosity - a scalar Dirichlet boundary condition for turbulent viscosity, υ (aka turbnubc, turbnu).

setids - One (or more) side-set(s) on which the boundary condition will be applied, setId (integer, required).

load_curve

A load curve allows for the specification of time-dependent boundary conditions. The load curve is defined as a list of time and factor pairs:

  • ti The time value
  • vi The load factor

name - A descriptive load curve name (string, default=null). name is not used in the derived control file. A unique id for each specific load curve is generated underneath the cover of the web interface.
