Turbulence

There are two ways to activate turbulence models. The first, turbulence model, presents a simplified way to specify a turbulence model, but does not permit the specification of any optional parameters for the model. All default values are used for the model parameter. The second approach, turbulence, permits the specification of optional parameters for a turbulence model.

Turbulence Model

The turbulence model activates and selects the specific turbulence model. Default turbulence model parameters are used. The default turbulence model parameters are presented in the theoretical discussion for each turbulence model in the Hydra-TH theory manual.

name - A descriptive turbulence model name (string, default=null). name is not used in the derived control file. A unique id for each specific turbulence model is generated underneath the cover of the web interface. While providing a descriptive name is not required, these descriptive names clearly increase usability.

tmodel - The turbulence model (string, default=No Turbulence Model). The tmodel must be one of the following:

  • No Turbulence Model - No explicit turbulence model is used,
  • Spalart-Allmaras Model - Activate the Spalart-Allmaras model,
  • Detached-Eddy Spalart-Allmaras Model - Activate the detached-eddy model based on Spalart-Allmaras,
  • Smagorinsky Subgrid-Scale Model - Use the Smagorinsky subgrid-scale model,
  • Wall-Adapted Large Eddy Subgrid-Scale Model - Use the wall-adapted large eddy subgrid-scale model,
  • RNG k-e Model - Activate the RNG k-ε model,
  • SST k-w Model - Activate the SST k-ω model (under development),
  • ksgs Subgrid-Scale Model - Use the ksgs subgrid-scale model (under development), or
  • LDKM ksgs Subgrid-Scale Model - Use the LDKM ksgs subgrid-scale model (under development).

Turbulence activates and selects the specific turbulence model. It allows the end-user to specify model-specific parameters. At this time, only selected turbulence models and model-specific parameters may be specified.

turbulence - The turbulence model (string, default=No Turbulence Model). The turbulence must be one of the following:

  • No Turbulence Model - No explicit turbulence model is used,
  • RNG k-e Model - Activate the RNG k-ε model,
  • Smagorinsky Subgrid-Scale Model - Use the Smagorinsky subgrid-scale model, or
  • Wall-Adapted Large Eddy Subgrid-Scale Model - Use the wall-adapted large eddy subgrid-scale model.

Turbulence RNG k-ε

Presents the RNG k-ε model-specific parameters.

timescale_limiter - Enable/disable time-scale limiting to control the over-production of turbulent kinetic energy in stagnation flows, timescale_limiter (string, default=off).

  • on - Enable time-scale limiter, or
  • off - Disable time-scale limiter.

Turbulence Smagorinsky

Presents the Smagorinsky model-specific parameters.

c_s - Smagorinsky model constant, Cs (float, default=0.18).

prandtl - Turbulent Prandtl number, Prt (float, default=0.8889).

schmidt - Turbulent Schmidt number, Sct (float, default=1.0).

Turbulence Wall-Adapted Large Eddy

Presents the Smagorinsky model-specific parameters.

c_w - WALE model constant, Cw (float, default=0.5).

prandtl - Turbulent Prandtl number, Prt (float, default=0.8889).

schmidt - Turbulent Schmidt number, Sct (float, default=1.0).
